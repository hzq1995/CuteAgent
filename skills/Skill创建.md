# Skill 创建流程

Skill 是放在 `skills/` 目录下的 Markdown 文件，用来向 Agent 传授特定领域的工作流程、背景知识或操作规范。Agent 在处理相关任务时可以主动读取对应的 Skill 文件，以获得更精准的指导。

---

## 1. 创建 Skill 文件

在 `skills/` 下新建一个 `.md` 文件，文件名使用中文或英文均可，建议与 Skill 主题保持一致，例如：

```
skills/发送钉钉消息.md
skills/数据分析流程.md
skills/MyWorkflow.md
```

Skill 文件**不会被自动加载**，只有当你明确读取它时内容才会进入上下文。每次需要使用某个 Skill 时，先用文件读取工具获取其内容，再按照其中的步骤执行。

---

## 2. Skill 文件的结构

一份好的 Skill 文件通常包含以下部分：

### 2.1 标题与一句话说明

```markdown
# 发送钉钉消息

通过 `send_dingtalk_message` 工具向钉钉群机器人发送通知。
```

### 2.2 使用场景（何时触发）

明确写出 **什么情况下应该使用这个 Skill**，让 Agent 能准确判断是否需要读取它。

```markdown
## 使用场景

- 用户要求发送钉钉消息或通知时。
- 任务执行结束需要汇报结果时。
- 定时任务触发了钉钉提醒时。
```

### 2.3 前置条件

列出执行该 Skill 前需要满足的环境或配置要求。

```markdown
## 前置条件

- 环境变量 `DINGTALK_WEBHOOK_URL` 已设置。
- 若启用了签名校验，`DINGTALK_ACCESS_TOKEN` 也需要配置。
```

### 2.4 分步操作流程

用有序列表写出具体步骤，每步尽量可独立执行或验证。

```markdown
## 操作步骤

1. 确认消息内容已准备好，避免发送空消息。
2. 调用 `send_dingtalk_message` 工具，传入 `content` 参数。
3. 检查工具返回值中是否包含 `"errcode": 0`，确认发送成功。
4. 若失败，根据 `errmsg` 字段排查原因（token 错误、频率限制等）。
```

### 2.5 示例（可选）

提供典型的输入输出或代码片段，降低理解门槛。

```markdown
## 示例

调用参数：
```json
{ "content": "今日报表已生成，请查收。" }
```

成功返回：
```json
{ "errcode": 0, "errmsg": "ok" }
```
```

### 2.6 注意事项与约束

写出容易踩的坑、安全注意事项或不应做的事。

```markdown
## 注意事项

- 每分钟最多发送 20 条消息，超出会被限流。
- 不要将敏感信息（密钥、密码）写入消息正文。
- `content` 字段不能超过 2000 个字符，超出请截断后分段发送。
```

---

## 3. 完整示例

以下是一份最小可用的 Skill 文件模板：

```markdown
# 示例 Skill

一句话说明这个 Skill 的用途。

## 使用场景

- 场景 A
- 场景 B

## 前置条件

- 条件 1

## 操作步骤

1. 步骤一
2. 步骤二
3. 步骤三

## 注意事项

- 注意点 1
```

---

## 4. 在 README 中登记

新增 Skill 后，建议在 `README.md` 中补充一行引用，方便其他开发者和 Agent 发现它：

```markdown
See `skills/发送钉钉消息.md` for the DingTalk message workflow.
```

---

## 5. 配套脚本

如果 Skill 描述的工作流需要辅助脚本支撑（例如数据预处理、格式转换、第三方 SDK 封装等），将脚本放在 `utils/` 目录下，而不是直接写在 Skill 文档中。

- `utils/` 中的脚本由 Agent 直接调用执行，而不是被其他模块 `import`。
- 脚本文件名应与 Skill 主题对应，例如 `utils/dingding_robot.py` 对应钉钉相关 Skill。
- Skill 文档中可以引用对应的 `utils` 模块路径，方便读者定位实现细节。

---

## 6. 约束

- Skill 文件只包含**文档**，不要在其中编写可执行代码逻辑。
- 每个 Skill 聚焦单一主题，不要把多个不相关的工作流合并到一个文件里。
- 描述要足够具体，让 Agent 无需猜测即可按步骤执行。
- 涉及外部服务时，务必在"前置条件"中说明所需的配置项。
