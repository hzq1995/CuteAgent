# 工具创建流程

CuteHarness 会在每次 agent turn 开始时扫描 `tools/*.py`。同一轮对话中的多次工具调用会使用同一份工具注册表；修改工具文件后，从下一轮对话开始生效。

## 1. 创建工具文件

在 `tools/` 下创建一个 Python 文件，例如 `tools/my_tool.py`。文件名建议和工具名一致，避免使用 `_` 开头，`_` 开头的文件不会被加载。

## 2. 声明工具定义

每个工具文件必须导出 `TOOL_DEFINITION`，格式和 OpenAI/DeepSeek function tool schema 一致：

```python
TOOL_DEFINITION = {
    "type": "function",
    "function": {
        "name": "my_tool",
        "description": "Describe when and how the agent should use this tool.",
        "parameters": {
            "type": "object",
            "properties": {
                "text": {"type": "string", "description": "Input text."},
            },
            "required": ["text"],
        },
    },
}
```

工具名必须唯一。重复工具名会让本轮 agent turn 失败并显示加载错误。

## 3. 实现 run 函数

每个工具文件必须导出 `run(context, **kwargs)`。`kwargs` 来自模型生成的工具参数。

```python
from typing import Any

from app.agent_tools import ToolContext


def run(context: ToolContext, text: str) -> dict[str, Any]:
    return {"echo": text, "workspace": str(context.base_dir)}
```

`context` 可用字段：

- `base_dir`: 项目根目录。
- `scheduled_tasks`: 定时任务存储。
- `memories`: 长期记忆存储。
- `task_store`: 会话存储。
- `python_timeout_seconds`: 设置页里的工具超时时间。
- `dingtalk_webhook_url`: DingTalk webhook。
- `dingtalk_access_token`: DingTalk access token。

## 4. 测试工具

新增工具后至少做两类检查：

```powershell
pytest tests/test_agent_tools.py
pytest
```

如果工具会执行外部命令，返回值建议包含 `stdout`、`stderr`、`exit_code`、`timed_out`，并使用 `app.agent_tools.truncate()` 截断长输出。

## 5. 约束

- 不要在工具 import 时执行副作用；把执行逻辑放进 `run()`。
- 不要依赖同一轮对话中重新加载工具；工具修改从下一轮对话开始生效。
- 让 `description` 写清楚使用场景，减少模型误调用。
